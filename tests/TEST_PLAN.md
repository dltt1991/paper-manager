## 1. 测试目标

### 1.1 总体目标

确保论文管理系统（Paper Manager）的功能符合产品需求文档（PRD）的要求，系统稳定可靠，用户体验良好。

### 1.2 具体目标

| 目标编号 | 目标描述 | 验收标准 |
|---------|---------|---------|
| TO-001 | 验证核心功能完整性 | 所有P0/P1功能测试用例通过率100% |
| TO-002 | 验证系统稳定性 | 连续运行24小时无严重错误 |
| TO-003 | 验证API接口正确性 | 所有API接口符合接口文档定义 |
| TO-004 | 验证数据一致性 | 数据库操作正确，无数据丢失 |
| TO-005 | 验证性能达标 | 页面加载时间<2s，搜索响应<300ms |

## 3. 测试策略

### 3.1 测试层次

```
┌─────────────────────────────────────────────┐
│           端到端测试 (E2E Tests)              │  ← 人工测试
│         验证完整业务流程                      │
├─────────────────────────────────────────────┤
│           集成测试 (Integration Tests)        │  ← 自动化测试
│         验证模块间交互                        │
├─────────────────────────────────────────────┤
│           单元测试 (Unit Tests)               │  ← 自动化测试
│         验证单个函数/组件                     │
└─────────────────────────────────────────────┘
```

### 3.2 测试方法

#### 3.2.1 后端API测试

- **框架**: pytest + httpx
- **策略**: 
  - 使用FastAPI TestClient进行集成测试
  - 使用SQLite内存数据库隔离测试数据
  - 使用Mock对象模拟Kimi API调用
  - 覆盖正常场景和异常场景

#### 3.2.2 前端组件测试

- **框架**: Vitest + React Testing Library（建议）
- **当前**: 以人工测试为主

#### 3.2.3 端到端测试

- **方式**: 人工测试
- **场景**: 
  - 完整的上传-查看-批注-删除流程
  - 边界条件测试（大文件、特殊字符等）

### 3.3 测试数据策略

| 数据类型 | 来源 | 说明 |
|---------|-----|------|
| 测试PDF文件 | fixtures/ | 包含正常PDF、损坏PDF、超大PDF |
| 测试数据库 | SQLite内存 | 每次测试独立初始化 |
| Mock数据 | 代码生成 | 模拟API响应数据 |
